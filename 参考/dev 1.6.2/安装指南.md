## 🔧 系统要求

### 必需环境

1. **SillyTavern**
   - 版本：最新稳定版
   - 下载地址：https://github.com/SillyTavern/SillyTavern

2. **JS-Slash-Runner 插件**
   - 这是 SillyTavern 的必需插件，提供 JavaScript 执行环境
   - 安装方法：在 SillyTavern 的扩展管理中搜索并安装 `JS-Slash-Runner`
   - 项目地址：https://github.com/N0VI028/JS-Slash-Runner

### 开发环境（仅完整安装需要）

- **Node.js**: >= 22.0.0
- **pnpm**: >= 10.0.0（推荐 10.19.0）

## ⚡ 快速安装（推荐）

> 适用于普通用户，无需编程知识

### 第一步：安装前置插件

1. 打开 SillyTavern
2. 进入 **扩展（Extensions）** 页面
3. 搜索并安装 **JS-Slash-Runner** 插件
4. 重启 SillyTavern 以激活插件

### 第二步：配置世界书

#### 2.1 创建系统提示词条目

1. 在 SillyTavern 中打开 **世界书（World Info）** 管理界面
2. 创建新的世界书条目，设置如下：
   - **条目名称**：`伊甸园系统提示词`
   - **深度（Depth）**：`0`（最高优先级）
   - **触发方式**：`Constant`（始终激活）
   - **内容**：复制 `系统提示词/系统提示词_完整版.md` 文件的全部内容

3. **（可选）** 如果 AI 生成的 YAML 格式不稳定，创建附属提示词条目：
   - **条目名称**：`YAML格式规范`
   - **深度（Depth）**：`0`
   - **触发方式**：`Constant`
   - **内容**：复制 `系统提示词/附属提示词_YAML格式规范.md` 文件的全部内容

#### 2.2 绑定世界书到角色卡

1. 在角色卡设置中，将刚才创建的世界书设置为 **主世界书（Primary Worldbook）**
2. 确保世界书条目已启用

### 第三步：配置正则表达式脚本

1. 在 SillyTavern 中打开 **正则表达式（Regex）** 管理界面
2. 创建新的正则表达式脚本：
   - **脚本名称**：`伊甸园前端界面`
   - **查找正则表达式**：`【\d+】`（匹配形如 `【00】`、`【01】` 的标记）
   - **替换内容**：将 `app/dist/index.html` 文件的全部内容用代码块包裹
   - **应用范围**：勾选 ✅ **AI 输出** 和 ✅ **用户输入**

**替换内容格式示例**：

```html
<!-- 将 app/dist/index.html 的完整内容粘贴在这里 -->
<!DOCTYPE html>
<html lang="zh-CN">
...
</html>
```

> **重要提示**：必须使用代码块（三个反引号）包裹 HTML 内容，否则正则替换将无法正确工作。

3. 保存并启用该正则表达式脚本

### 第四步：激活系统

1. 在角色卡的 **开场白（First Message）** 或第一条消息中添加激活标记：`【00】`
2. 开始对话，系统将自动加载前端界面

## 🛠️ 完整安装（开发者）

> 适用于需要自定义或调试前端应用的开发者

### 第一步：克隆或下载项目

```bash
# 克隆仓库
git clone <repository-url>
cd SillyTavern-tongceng

# 或直接解压源代码压缩包
```

### 第二步：安装依赖

```bash
# 全局安装 pnpm（如果尚未安装）
npm install -g pnpm

# 进入前端应用目录
cd app

# 安装项目依赖
pnpm install
```

### 第三步：开发或构建

#### 开发模式（实时预览）

```bash
# 启动开发服务器
pnpm run dev
```

开发服务器将在 `http://localhost:5000` 启动，支持热重载。

#### 构建生产版本

```bash
# 构建前端应用（包含类型检查）
pnpm run build
```

构建产物将输出到 `app/dist/` 目录，生成单文件 `index.html`。

#### 其他有用命令

```bash
# 类型检查
pnpm run type-check

# 代码检查（自动修复）
pnpm run lint

# 代码检查（仅检查不修复）
pnpm run lint:check

# 代码格式化
pnpm run format

# 代码格式化检查
pnpm run format:check

# 运行测试（监听模式）
pnpm run test

# 运行测试（单次运行）
pnpm run test:run

# 运行测试（带覆盖率）
pnpm run test:coverage

# 运行测试（UI 模式）
pnpm run test:ui

# 生成 API 文档
pnpm run docs

# 分析依赖关系
pnpm run analyze:deps
```

### 第四步：配置 SillyTavern

完成构建后，按照 [快速安装](#快速安装推荐) 中的步骤 2-4 配置 SillyTavern。

## 🎮 SillyTavern 配置

### 世界书配置详解

#### 系统提示词条目

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **条目名称** | `伊甸园系统提示词` | 可自定义 |
| **深度（Depth）** | `0` | 最高优先级，确保 AI 始终遵循系统规则 |
| **触发方式** | `Constant` | 始终激活，无需关键词触发 |
| **插入位置** | `After Character Defs` | 在角色定义之后插入 |
| **角色** | `System` | 作为系统消息发送 |

#### 附属提示词条目（可选）

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **条目名称** | `YAML格式规范` | 可自定义 |
| **深度（Depth）** | `0` | 与主提示词相同优先级 |
| **触发方式** | `Constant` | 始终激活 |
| **用途** | 强化 YAML 格式输出规范 | 当 AI 输出格式不稳定时启用 |

### 正则表达式配置详解

#### 前端界面注入脚本

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **脚本名称** | `伊甸园前端界面` | 可自定义 |
| **查找正则** | `【\d+】` | 匹配 `【00】`、`【01】` 等标记 |
| **替换内容** | ` ```html\n<完整的 index.html 内容>\n``` ` | 必须用代码块包裹 |
| **应用范围** | ✅ AI 输出 + ✅ 用户输入 | 双向应用 |
| **执行顺序** | 默认 | 无特殊要求 |

> **技巧**：查找正则表达式可以自定义，只要在开场白中使用相同的标记即可。例如使用 `\[EDEN\]` 作为触发标记。

## ❓ 常见问题

### Q1: 前端界面没有显示怎么办？

**可能原因**：
1. JS-Slash-Runner 插件未安装或未启用
2. 正则表达式脚本配置错误
3. 开场白中缺少激活标记（如 `【00】`）

**解决方法**：
1. 检查 SillyTavern 扩展页面，确认 JS-Slash-Runner 已安装并启用
2. 检查正则表达式脚本的 "应用范围" 是否勾选了 "AI 输出" 和 "用户输入"
3. 确保 `app/dist/index.html` 的内容已正确复制到替换内容中，并用代码块包裹
4. 在角色卡开场白或第一条消息中添加 `【00】` 标记

### Q2: AI 返回的数据格式错误怎么办？

**可能原因**：
1. 系统提示词未正确配置
2. AI 模型不稳定或理解能力不足

**解决方法**：
1. 确认世界书中的系统提示词条目已启用，深度设置为 `0`
2. 启用附属提示词 `系统提示词/附属提示词_YAML格式规范.md`
3. 尝试使用更强大的 AI 模型（如 GPT-4、Claude 3.5 等）
4. 检查 AI 返回的消息，确保包含 ` ```yaml ... ``` ` 代码块

### Q3: 如何更新前端应用？

**快速安装用户**：
1. 下载最新版本的 `app/dist/index.html`
2. 更新正则表达式脚本中的替换内容

**完整安装用户**：
1. 拉取最新代码：`git pull`
2. 重新安装依赖：`cd app && pnpm install`
3. 重新构建：`pnpm run build`
4. 更新正则表达式脚本中的替换内容

### Q4: 世界书数据如何管理？

系统会自动将游戏数据保存到世界书中：
- **自动存档**：每次游戏状态变化时自动保存到世界书条目 `自动存档数据`
- **手动存档**：在游戏界面中可以创建多个命名存档
- **初始化数据**：首次初始化的数据保存在 `初始化数据` 条目中

所有存档数据以 YAML 格式存储在世界书中，可以手动编辑或导出。

### Q5: 如何自定义游戏内容？

1. **修改初始数据**：编辑 `app/public/data.yaml` 文件，定义角色、物品、成就等初始数据
2. **调整系统提示词**：修改 `系统提示词/系统提示词_完整版.md`，自定义 AI 行为和剧情风格
3. **自定义界面**：修改 `app/src/` 目录下的 Vue 组件，重新构建后更新正则脚本

### Q6: 支持哪些浏览器？

前端应用基于现代 Web 技术构建，支持：
- ✅ Chrome/Edge（推荐）
- ✅ Firefox
- ✅ Safari
- ❌ IE（不支持）

建议使用最新版本的浏览器以获得最佳体验。

### Q7: 如何使用示例角色卡？

如果项目提供了示例角色卡：
1. 将示例角色卡导入到 SillyTavern
2. 示例角色卡已预配置世界书条目和正则表达式
3. 将世界书设置为全局（Global），即可在所有角色卡中使用
4. 直接开始对话，无需额外配置
