# Eden System - 状态管理说明

## 1. 概述

Eden System 使用 [Pinia](https://pinia.vuejs.org/) 作为状态管理库。Pinia 是 Vue 官方推荐的状态管理解决方案，相比 Vuex 更加轻量、类型安全且易于使用。

## 2. Store 架构

应用包含三个主要的 Store：

```
Pinia
├── gameStore        # 游戏核心数据
├── navigationStore  # 页面导航状态
└── settingsStore    # 应用设置
```

## 3. gameStore（游戏数据 Store）

**位置**: `src/stores/gameStore.ts`

管理游戏的核心数据状态，包括配置、故事、角色、商店、成就等。

### 3.1 状态（State）

<augment_code_snippet path="app/src/stores/gameStore.ts" mode="EXCERPT">

```typescript
const gameData = ref<GameData>({})
const userActionLog = ref<UserAction[]>([])
const isLoading = ref(true)
const loadError = ref<string | null>(null)
const needsInitialization = ref(false)
```

</augment_code_snippet>

**字段说明**:

- `gameData`: 游戏核心数据对象
- `userActionLog`: 用户操作日志（用于 AI Prompt）
- `isLoading`: 数据加载状态
- `loadError`: 加载错误信息
- `needsInitialization`: 是否需要初始化

### 3.2 计算属性（Getters）

从 `gameData` 中提取各个子对象：

```typescript
const config = computed(() => gameData.value.config)
const story = computed(() => gameData.value.story)
const choices = computed(() => gameData.value.choices)
const characters = computed(() => gameData.value.characters)
const shop = computed(() => gameData.value.shop)
const storage = computed(() => gameData.value.storage)
const achievements = computed(() => gameData.value.achievements)
const summaries = computed(() => gameData.value.summaries)
```

### 3.3 操作（Actions）

#### loadData()

加载游戏数据。

```typescript
async function loadData(): Promise<void> {
  isLoading.value = true
  loadError.value = null

  const result = await GameDataService.loadGameData()

  gameData.value = result.gameData
  needsInitialization.value = result.needsInitialization
  loadError.value = result.error
  isLoading.value = false
}
```

#### updateGameDataFromAI()

从 AI 响应更新游戏数据。

```typescript
function updateGameDataFromAI(yamlContent: string): void {
  GameDataService.updateGameDataFromAI(gameData.value, yamlContent)
}
```

**说明**: 接收 AI 返回的 YAML 内容（包含 `$update` 和 `$delete` 操作符），应用到游戏数据。

#### setGameData()

设置游戏数据（用于从存档加载或导入数据）。

```typescript
function setGameData(data: GameData): void {
  gameData.value = data
}
```

#### resetGameData()

重置游戏数据到空状态。

```typescript
function resetGameData(): void {
  gameData.value = {}
  userActionLog.value = []
  needsInitialization.value = true
}
```

#### logUserAction()

记录用户操作（用于 AI Prompt）。

```typescript
function logUserAction(action: UserAction): void {
  userActionLog.value.push(action)
}
```

#### clearUserActionLog()

清空用户操作日志。

```typescript
function clearUserActionLog(): void {
  userActionLog.value = []
}
```

#### autoSaveToWorldbook()

自动保存游戏数据到世界书（仅在自动保存开关开启时执行）。

```typescript
async function autoSaveToWorldbook(): Promise<void> {
  const settingsStore = useSettingsStore()
  if (!settingsStore.settings.autoSave) return
  await GameDataService.autoSaveToWorldbook(gameData.value)
}
```

#### exportGameData()

导出游戏数据为 YAML 字符串。

```typescript
function exportGameData(): string {
  return GameDataService.exportGameData(gameData.value, userActionLog.value.length)
}
```

#### $reset()

重置 Store 到初始状态。

```typescript
function $reset(): void {
  gameData.value = {}
  userActionLog.value = []
  isLoading.value = true
  loadError.value = null
  needsInitialization.value = false
}
```

### 3.4 使用示例

```typescript
import { useGameStore } from '@/stores/gameStore'
import { PathResolver } from '@/services/dataUpdate'

const gameStore = useGameStore()

// 读取状态
const currentStory = gameStore.story
const playerCharacter = gameStore.characters?.player

// 直接修改响应式数据（简单更新）
gameStore.gameData.shop.currency = 1000

// 使用 PathResolver 更新（复杂路径）
PathResolver.setValueByPath(gameStore.gameData, 'shop.currency', 1000)

// 从 AI 响应更新数据
const yamlContent = `
$update:
  "gameData.shop.currency": 1000
  "gameData.characters.player.level": 2
`
gameStore.updateGameDataFromAI(yamlContent)

// 加载数据
await gameStore.loadData()

// 记录用户操作
gameStore.logUserAction({
  type: 'choice',
  content: '选择了选项1',
  timestamp: Date.now(),
})

// 自动保存到世界书
await gameStore.autoSaveToWorldbook()
```

## 4. navigationStore（导航状态 Store）

**位置**: `src/stores/navigationStore.ts`

管理页面导航状态和滚动行为。

### 4.1 状态（State）

```typescript
const currentPage = ref('home')
```

**字段说明**:

- `currentPage`: 当前页面 ID（home、status、shop、cart、storage、achievements、review、load、settings）

### 4.2 操作（Actions）

#### navigateTo()

导航到指定页面。

```typescript
function navigateTo(pageId: string): void {
  currentPage.value = pageId
  scrollToTop()
}
```

#### scrollToTop()

滚动到页面顶部。

```typescript
function scrollToTop(): void {
  const activePage = document.querySelector('.page.active')
  if (activePage) {
    const pageTitle = activePage.querySelector('#page-title')
    if (pageTitle) {
      pageTitle.scrollIntoView({
        behavior: 'smooth',
        block: 'start',
        inline: 'nearest',
      })
    }
  }
}
```

#### $reset()

重置 Store 到初始状态。

```typescript
function $reset(): void {
  currentPage.value = 'home'
}
```

### 4.3 使用示例

```typescript
import { useNavigationStore } from '@/stores/navigationStore'

const navigationStore = useNavigationStore()

// 读取当前页面
const page = navigationStore.currentPage

// 导航到商店页
navigationStore.navigateTo('shop')

// 滚动到顶部
navigationStore.scrollToTop()
```

### 4.4 注意事项

- 页面导航状态**不持久化**，每次刷新都从主页开始
- 导航时自动滚动到页面顶部

## 5. settingsStore（应用设置 Store）

**位置**: `src/stores/settingsStore.ts`

管理应用的全局设置，并自动持久化到 localStorage。

### 5.1 状态（State）

```typescript
const settings = ref<AppSettings>({
  maxRetries: 3,
  maxDataRetries: 3,
  retryDelay: 1000,
  debugMode: false,
  autoSave: true,
})
```

**AppSettings 接口**:

```typescript
interface AppSettings {
  maxRetries: number // AI 请求最大重试次数
  maxDataRetries: number // 数据加载最大重试次数
  retryDelay: number // 重试延迟（毫秒）
  debugMode: boolean // 调试模式
  autoSave: boolean // 自动保存
}
```

### 5.2 操作（Actions）

#### loadSettings()

从 localStorage 加载设置。

```typescript
function loadSettings(): void {
  try {
    const savedSettings = localStorage.getItem('eden-system-settings')
    if (savedSettings) {
      const parsed = JSON.parse(savedSettings)
      settings.value = { ...DEFAULT_SETTINGS, ...parsed }
    }
  } catch (error) {
    logger.error('加载设置失败:', error)
    settings.value = { ...DEFAULT_SETTINGS }
  }
}
```

#### saveSettings()

保存设置到 localStorage。

```typescript
function saveSettings(): void {
  try {
    localStorage.setItem('eden-system-settings', JSON.stringify(settings.value))
  } catch (error) {
    logger.error('保存设置失败:', error)
  }
}
```

#### resetSettings()

重置设置为默认值。

```typescript
function resetSettings(): void {
  settings.value = { ...DEFAULT_SETTINGS }
  saveSettings()
}
```

#### updateSetting()

更新单个设置项。

```typescript
function updateSetting<K extends keyof AppSettings>(key: K, value: AppSettings[K]): void {
  settings.value[key] = value
  saveSettings()
}
```

#### updateSettings()

批量更新设置。

```typescript
function updateSettings(newSettings: Partial<AppSettings>): void {
  settings.value = { ...settings.value, ...newSettings }
  saveSettings()
}
```

### 5.3 自动保存

settingsStore 使用 Vue 的 `watch` 监听设置变化，自动保存到 localStorage：

```typescript
watch(
  settings,
  () => {
    saveSettings()
  },
  { deep: true }
)
```

### 5.4 使用示例

```typescript
import { useSettingsStore } from '@/stores/settingsStore'

const settingsStore = useSettingsStore()

// 读取设置
const maxRetries = settingsStore.settings.maxRetries
const debugMode = settingsStore.settings.debugMode

// 更新单个设置
settingsStore.updateSetting('debugMode', true)

// 批量更新设置
settingsStore.updateSettings({
  maxRetries: 5,
  retryDelay: 2000,
})

// 重置设置
settingsStore.resetSettings()
```

## 8. 数据持久化策略

| Store           | 持久化 | 存储位置           | 时机               |
| --------------- | ------ | ------------------ | ------------------ |
| gameStore       | 是     | SillyTavern 世界书 | 玩家操作后自动保存 |
| navigationStore | 否     | -                  | 不持久化           |
| settingsStore   | 是     | localStorage       | 设置变化时自动保存 |

## 9. 最佳实践

### 9.1 状态访问

- 优先使用计算属性（Getters）而非直接访问嵌套状态
- 使用 `storeToRefs` 解构状态以保持响应性
- 避免在组件外部直接修改 Store 状态

### 9.2 状态更新

- 始终通过 Actions 更新状态
- 使用 `GameDataUpdater` 更新游戏数据
- 批量更新使用 `batchUpdate` 提高性能

### 9.3 类型安全

- 充分利用 TypeScript 类型推导
- 为 Actions 添加类型注解
- 使用接口定义状态结构

### 9.4 性能优化

- 避免在计算属性中进行复杂计算
- 使用 `watch` 时注意性能影响
- 大数据更新使用批量操作

## 11. 常见问题

**Q: 为什么解构的状态失去了响应性？**

A: 使用 `storeToRefs` 解构状态以保持响应性：

```typescript
// ❌ 错误：失去响应性
const { gameData } = gameStore

// ✅ 正确：保持响应性
const { gameData } = storeToRefs(gameStore)
```

**Q: 如何在 Store 外部访问其他 Store？**

A: 在 Action 中直接导入并使用其他 Store：

```typescript
import { useSettingsStore } from './settingsStore'

function someAction() {
  const settingsStore = useSettingsStore()
  const maxRetries = settingsStore.settings.maxRetries
  // ...
}
```

**Q: 如何重置所有 Store？**

A: 调用每个 Store 的 `$reset()` 方法：

```typescript
const gameStore = useGameStore()
const navigationStore = useNavigationStore()
const settingsStore = useSettingsStore()

gameStore.$reset()
navigationStore.$reset()
settingsStore.resetSettings()
```
