# Eden System - 架构设计说明

## 1. 整体架构

Eden System 采用现代化的前端架构，基于 Vue 3 生态系统构建，专为 SillyTavern 的 iframe 环境设计。

## 2. 目录结构和模块划分

### 2.1 核心模块

#### Components（组件层）

- **common/**: 通用 UI 组件，可在多个页面复用
  - 输入组件：CustomNumberInput、CustomSelect、QuantitySelector
  - 反馈组件：LoadingSpinner、ToastContainer、EmptyState
  - 导航组件：Pagination、ScrollToTopButton
  - 其他：ProgressBar、SafeImage、SkeletonCard

- **layout/**: 布局组件，定义应用结构
  - NavBar: 顶部导航栏
  - PageRouter: 页面路由器（自定义路由系统）
  - Modal: 模态框
  - AILoadingOverlay: AI 加载遮罩

- **pages/**: 页面组件，对应不同的功能页面
  - HomePage: 主页（故事和选择）
  - StatusPage: 状态页（角色信息）
  - ShopPage: 商店页
  - CartPage: 购物车页
  - StoragePage: 物品存储页
  - AchievementsPage: 成就页
  - ReviewPage: 剧情回顾页
  - LoadGamePage: 存档管理页
  - SettingsPage: 设置页
  - InitializationPage: 初始化页

#### Composables（组合式函数层）

提供可复用的业务逻辑，遵循 Vue 3 Composition API 最佳实践，按功能分类组织：

- **achievement/**: 成就相关
  - useAchievementProgress: 成就进度计算
  - useAchievementUnlock: 成就自动解锁
- **ai/**: AI 通信相关
  - useAICommunication: AI 通信
  - useAILoadingState: AI 加载状态
- **app/**: 应用生命周期相关
  - useAILoadingOverlay: AI 加载遮罩
  - useAppErrorHandling: 应用错误处理
  - useAppLifecycle: 应用生命周期
  - useHeightSync: iframe 高度同步
- **form/**: 表单验证相关
  - useFormValidation: 表单验证
- **game/**: 游戏逻辑相关
  - useGameInitialization: 游戏初始化
  - useItemEffects: 物品效果应用
  - usePlayerChoice: 玩家选择处理
  - useSearch: 搜索功能
  - useStorageFiltering: 存储筛选
- **save/**: 存档系统
  - useSaveManagement: 存档管理
  - useSaveOperations: 存档操作
  - useSaveList: 存档列表
- **shop/**: 商店系统
  - useCart: 购物车基础逻辑
  - useCartCheckout: 购物车结算
  - useCartCoupon: 优惠券逻辑
  - useShop: 商店逻辑
  - useShopFilters: 商店筛选
  - useShoppingCart: 购物车管理
- **ui/**: UI 交互相关
  - useFullscreen: 全屏控制
  - useHtmlSanitizer: HTML 安全处理
  - useIframeHeightSync: iframe 高度同步
  - useModal: 模态框控制
  - usePageNavigation: 页面导航
  - usePagination: 分页逻辑
  - useStoryContentProcessor: 故事内容处理
  - useTheme: 主题控制
  - useToast: Toast 通知
  - useTTS: 文本转语音

#### Services（服务层）

封装业务逻辑和外部 API 调用：

- **api/**: API 调用封装
  - aiApi.ts: AI 生成 API
  - characterApi.ts: 角色卡 API
  - minimaxApi.ts: MiniMax TTS API
  - worldbookApi.ts: 世界书 API
  - index.ts: API 统一导出

- **worldbook/**: 世界书服务
  - worldbookConnectionService.ts: 世界书连接管理
  - worldbookDataService.ts: 世界书数据读写
  - worldbookSaveService.ts: 存档管理
  - index.ts: 世界书服务统一导出

- **dataUpdate/**: 数据更新服务
  - dataMerger.ts: 数据合并器
  - pathResolver.ts: 路径解析器
  - yamlParser.ts: YAML 解析器
  - index.ts: 数据更新统一导出

- **核心服务**:
  - gameDataService.ts: 游戏数据加载和初始化
  - aiService.ts: AI 服务（发送请求、重试机制、响应解析）
  - dataBackupService.ts: 数据备份服务
  - baseCacheService.ts: 缓存服务基类
  - imageCacheService.ts: 图片缓存服务
  - stChatu8ImageService.ts: SillyTavern Chatu8 图片服务
  - ttsCacheService.ts: TTS 缓存服务

**注**: aiResponseParser.ts 已废弃，其逻辑已合并到 aiService.ts 中

#### Stores（状态管理层）

使用 Pinia 管理全局状态：

- **gameStore**: 游戏核心数据（config、story、characters、shop、achievements 等）
- **navigationStore**: 页面导航状态
- **settingsStore**: 应用设置（重试次数、调试模式、自动保存等）

#### Types（类型定义层）

TypeScript 类型定义，按功能分类：

- **api.ts**: API 相关类型
- **common.ts**: 通用类型
- **config.ts**: 配置类型
- **gameData.ts**: 游戏数据类型
- **shop.ts**: 商店相关类型
- **ui.ts**: UI 相关类型
- **utility.ts**: 工具类型
- **external-apis.d.ts**: 外部 API 类型声明

#### Utils（工具函数层）

纯函数工具，无副作用：

- **dataValidation.ts**: 数据验证
- **dateUtils.ts**: 日期处理
- **debounce.ts**: 防抖函数
- **environment.ts**: 环境检测
- **errorHandler.ts**: 错误处理
- **logger.ts**: 日志工具
- **pathUtils.ts**: 路径工具
- **sanitize.ts**: 安全处理
- **storage.ts**: 本地存储
- **typeGuards/**: 类型守卫

## 3. 数据流向

### 3.1 数据加载流程

```
应用启动
    │
    ▼
检查 JS-Slash-Runner 环境
    │
    ├─► 不可用 ──► 加载本地示例数据 (public/data.yaml)
    │
    ▼
检查世界书绑定
    │
    ├─► 未绑定 ──► 提示绑定世界书
    │
    ▼
检查世界书数据
    │
    ├─► 有自动存档 ──► 加载自动存档
    ├─► 有初始化数据 ──► 加载初始化数据
    └─► 无数据 ──► 显示初始化页面
```

### 3.2 玩家选择流程

```
玩家点击选择
    │
    ▼
usePlayerChoice.handleChoice()
    │
    ▼
useAICommunication.sendPlayerChoice()
    │
    ├─► 构建 Prompt（包含游戏数据、选择、历史）
    ├─► 调用 AIApi.generate()
    ├─► 解析 AI 响应（YAML 格式）
    └─► 重试机制（失败时）
    │
    ▼
更新游戏数据
    │
    ├─► GameDataUpdater.applyUpdates()
    ├─► 更新 gameStore
    └─► 自动保存到世界书
    │
    ▼
触发成就检查
    │
    └─► useAchievementUnlock（自动解锁）
```

### 3.3 存档流程

```
手动保存
    │
    ▼
WorldbookSaveService.saveGame()
    │
    ├─► 序列化游戏数据为 YAML
    ├─► 创建/更新世界书条目
    └─► 保存到世界书

加载存档
    │
    ▼
WorldbookSaveService.loadSave()
    │
    ├─► 从世界书读取存档条目
    ├─► 解析 YAML 数据
    └─► 更新 gameStore
```

## 5. 设计模式

### 5.1 Composition API 模式

使用 Vue 3 的 Composition API 组织代码，将相关逻辑封装到 Composables 中：

- **单一职责**: 每个 Composable 只负责一个功能
- **可复用性**: Composables 可在多个组件中复用
- **类型安全**: 充分利用 TypeScript 类型推导

### 5.2 服务层模式

将业务逻辑从组件中分离到 Services 层：

- **关注点分离**: 组件只负责 UI，业务逻辑在 Services
- **可测试性**: Services 是纯函数或类，易于单元测试
- **可维护性**: 业务逻辑集中管理，易于修改

### 5.3 状态管理模式

使用 Pinia 管理全局状态：

- **模块化**: 按功能划分 Store（game、navigation、settings）
- **响应式**: 利用 Vue 的响应式系统
- **持久化**: settingsStore 自动持久化到 localStorage

### 5.4 错误处理模式

统一的错误处理机制：

- **ErrorBoundary**: 捕获组件错误
- **AppError**: 自定义错误类，包含错误分类和用户友好消息
- **Toast 通知**: 向用户展示错误信息
- **日志记录**: 使用 logger 记录错误详情

### 5.5 重试模式

AI 请求和数据加载的重试机制：

- **指数退避**: 重试间隔逐渐增加
- **最大重试次数**: 可配置的重试上限
- **用户反馈**: 显示重试进度和原因

## 6. 性能优化

### 6.1 构建优化

- **单文件构建**: 使用 vite-plugin-singlefile 生成单个 HTML 文件
- **代码压缩**: Terser 压缩，兼容性好
- **Tree Shaking**: 移除未使用的代码
- **外部依赖**: Vue、Pinia 等通过全局变量引入，减小包体积

### 6.2 运行时优化

- **虚拟滚动**: 大列表使用分页而非虚拟滚动（简化实现）
- **防抖**: 搜索、输入等使用防抖减少计算
- **计算属性缓存**: 利用 Vue 的计算属性缓存
- **组件懒加载**: 页面组件按需加载（通过 PageRouter）

### 6.3 内存优化

- **及时清理**: 组件卸载时清理定时器、监听器
- **避免内存泄漏**: 使用 WeakMap、WeakSet 存储临时数据
- **数据结构优化**: 使用合适的数据结构（Map、Set）

## 7. 安全性

### 7.1 XSS 防护

- **DOMPurify**: 所有用户输入和 AI 生成的 HTML 都经过 DOMPurify 处理
- **禁用 v-html**: ESLint 规则禁止使用 `v-html`（除非必要且经过安全处理）

### 7.2 数据验证

- **类型守卫**: 运行时类型检查
- **数据验证**: 使用 dataValidation.ts 验证数据结构
- **错误处理**: 捕获并处理无效数据

### 7.3 权限控制

- **环境检测**: 检查 JS-Slash-Runner 环境是否可用
- **API 可用性检查**: 调用 API 前检查是否可用
- **降级处理**: API 不可用时使用本地数据

## 8. 可扩展性

### 8.1 添加新页面

1. 在 `src/components/pages/` 创建页面组件
2. 在 `PageRouter.vue` 中注册页面
3. 在 `config.navButtons` 中添加导航按钮（如需要）

### 8.2 添加新功能

1. 在 `src/types/` 定义类型
2. 在 `src/services/` 实现业务逻辑
3. 在 `src/composables/` 创建 Composable
4. 在组件中使用 Composable

### 8.3 扩展数据模型

1. 在 `src/types/gameData.ts` 中扩展 `GameData` 接口
2. 更新相关的 Services 和 Composables
3. 更新 UI 组件以展示新数据
