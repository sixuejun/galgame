# Eden System - 数据模型说明

## 1. 概述

Eden System 的数据模型采用 YAML 格式存储，通过 TypeScript 接口定义类型。所有数据模型定义在 `src/types/` 目录下。

## 2. 核心数据结构

### 2.1 GameData（游戏数据根对象）

游戏的根数据对象，包含所有游戏状态。

**类型定义**: `src/types/gameData.ts`

```typescript
interface GameData {
  config?: Config // 游戏配置
  story?: Story // 当前故事状态
  choices?: Choice[] // 玩家可选择的选项
  characters?: {
    // 角色数据（键为角色 ID）
    [characterId: string]: Character
  }
  shop?: ShopData // 商店数据
  storage?: StorageData // 物品存储数据
  achievements?: {
    // 成就数据（键为成就 ID）
    [achievementId: string]: Achievement
  }
  summaries?: Summary[] // 剧情摘要记录
}
```

**示例**:

```yaml
gameData:
  config:
    phase: 渗透期
    version: '1.3'
  story:
    time: '2024-01-01 10:00'
    location: '咖啡厅'
```

### 2.2 Config（配置数据）

游戏的配置信息，包括版本、阶段、导航按钮、页面配置等。

**类型定义**: `src/types/config.ts`

```typescript
interface Config {
  version: string // 游戏版本
  phase: string // 当前阶段
  navButtons?: {
    // 导航按钮配置
    [key: string]: {
      name?: string // 按钮名称
      icon?: string // 按钮图标（Font Awesome）
    }
  }
  cart?: CartConfig // 购物车配置
  home?: HomePageConfig // 主页配置
  status?: StatusPageConfig // 状态页配置
  shop?: PageConfig // 商店页配置
  storage?: PageConfig // 存储页配置
  achievements?: PageConfig // 成就页配置
  review?: PageConfig // 回顾页配置
}
```

**字段说明**:

- `version`: 游戏数据版本号，用于兼容性检查
- `phase`: 游戏当前阶段（如"渗透期"、"控制期"等）
- `navButtons`: 导航栏按钮配置，键为页面 ID（home、status、shop 等）

### 2.6 ShopData（商店数据）

商店的货币和物品信息。

**类型定义**: `src/types/shop.ts`

```typescript
interface ShopData {
  currency: number // 当前货币数量
  currencyType?: string // 货币类型（符号或名称）
  currencyDisplayMode?: CurrencyDisplayMode // 显示模式
  items: {
    // 商店物品（键为物品 ID）
    [itemId: string]: Item
  }
}

type CurrencyDisplayMode = 'symbol' | 'name'
```

**字段说明**:

- `currency`: 玩家当前拥有的货币数量
- `currencyType`: 货币类型，如 "¥"、"金币"、"积分" 等
- `currencyDisplayMode`:
  - `'symbol'`: 货币类型在前（如 "¥100"）
  - `'name'`: 数值在前（如 "100 金币"）

### 2.7 Item（物品）

商店或存储中的物品。

**类型定义**: `src/types/shop.ts`

```typescript
interface Item {
  id: string // 物品 ID
  name: string // 物品名称
  icon: string // 物品图标（Font Awesome）
  description: string // 物品描述
  price: number // 物品价格
  category: string // 物品分类
  quantity?: number // 数量（存储中使用）
  effects?: {
    // 物品效果（键为数据路径）
    [path: string]: ItemEffect
  }
  coupon?: CouponEffect // 优惠券效果
}
```

**ItemEffect（物品效果）**:

```typescript
interface ItemEffect {
  operation: 'add' | 'subtract' | 'set' | 'add_status' | 'remove_status'
  value: number | string | Record<string, unknown>
  description?: string // 效果描述
  maxPath?: string // 最大值路径（用于 add 操作）
}
```

**效果类型说明**:

- `add`: 增加数值（如增加货币、增加属性）
- `subtract`: 减少数值
- `set`: 设置为指定值
- `add_status`: 添加状态（如添加 trait）
- `remove_status`: 移除状态

**示例**:

```yaml
shop:
  currency: 1000
  currencyType: '金币'
  currencyDisplayMode: 'name'
  items:
    item_001:
      id: 'item_001'
      name: '恢复药水'
      icon: 'fa-flask'
      description: '恢复 50 点理智值'
      price: 100
      category: '消耗品'
      effects:
        'characters.player.psychologicalState.content.理智.current':
          operation: 'add'
          value: 50
          maxPath: 'characters.player.psychologicalState.content.理智.max'
```

### 2.9 Achievement（成就）

成就数据，包括解锁状态和进度条件。

**类型定义**: `src/types/gameData.ts`

```typescript
interface Achievement {
  id: string // 成就 ID
  name: string // 成就名称
  desc: string // 成就描述
  icon: string // 成就图标（Font Awesome）
  unlocked: boolean // 是否已解锁
  date?: string | null // 解锁日期（ISO 8601 格式）
  rarity?: string // 稀有度（如 "普通"、"稀有"、"史诗"）
  schedule?: AchievementSchedule // 进度条件
}
```

**AchievementSchedule（成就进度条件）**:

```typescript
interface AchievementSchedule {
  [dataPath: string]: AchievementCondition | number
}

interface AchievementCondition {
  desc: string // 条件描述
  target: number // 目标值
}
```

**示例**:

```yaml
achievements:
  ach_001:
    id: 'ach_001'
    name: '初次见面'
    desc: '第一次与ナナミ对话'
    icon: 'fa-handshake'
    unlocked: true
    date: '2024-01-01T10:00:00Z'
    rarity: '普通'
  ach_002:
    id: 'ach_002'
    name: '富有'
    desc: '拥有 10000 金币'
    icon: 'fa-coins'
    unlocked: false
    schedule:
      'shop.currency':
        desc: '当前金币'
        target: 10000
```

## 5. 数据路径规范

数据路径用于定位和更新游戏数据中的特定字段，采用点分隔的字符串格式。

**格式**: `对象.属性.子属性...`

**示例**:

- `shop.currency` - 商店货币
- `characters.player.level` - 玩家等级
- `characters.player.psychologicalState.content.理智.current` - 玩家理智值
- `achievements.ach_001.unlocked` - 成就解锁状态

**使用场景**:

1. **物品效果**: 定义物品对哪个数据产生影响
2. **成就进度**: 定义成就的进度条件
3. **数据更新**: 通过 `PathResolver` 或 `DataMerger` 更新数据

## 7. 数据更新机制

### 7.1 PathResolver

用于通过路径操作游戏数据的工具类。

**位置**: `src/services/dataUpdate/pathResolver.ts`

**主要方法**:

- `setValueByPath(obj, path, value)`: 通过路径设置值
- `deleteByPath(obj, path)`: 通过路径删除属性
- `getValueByPath(obj, path)`: 通过路径获取值

### 7.2 DataMerger

用于应用 YAML 更新操作的工具类。

**位置**: `src/services/dataUpdate/dataMerger.ts`

**主要方法**:

- `applyYamlUpdate(gameData, yamlContent)`: 应用 YAML 更新（支持 `$update` 和 `$delete` 操作符）

**YAML 更新格式**:

```yaml
$update:
  'gameData.shop.currency': 1000
  'gameData.characters.player.level': 2

$delete:
  - 'gameData.system.storage.inventory.used_item'
```

### 7.3 YamlParser

用于解析 YAML 格式数据的工具。

**位置**: `src/services/dataUpdate/yamlParser.ts`

**主要方法**:

- `parseYaml(yamlString)`: 解析 YAML 字符串为对象
- `extractYamlCodeBlock(text)`: 从文本中提取 YAML 代码块

## 9. 数据迁移

当数据模型发生变化时，需要进行数据迁移以保持兼容性。

**版本控制**: 通过 `config.version` 字段标识数据版本

**迁移策略**:

1. 检查数据版本
2. 应用必要的迁移脚本
3. 更新版本号

## 10. 最佳实践

### 10.1 类型安全

- 始终使用 TypeScript 接口定义数据结构
- 使用类型守卫进行运行时类型检查
- 避免使用 `any` 类型

### 10.2 数据验证

- 加载数据后立即验证
- 更新数据前验证新值
- 提供友好的错误提示

### 10.3 数据更新

- 简单更新可以直接修改响应式数据（Vue 3 会自动追踪）
- 复杂更新使用 `PathResolver` 或 `DataMerger`
- AI 返回的更新使用 `DataMerger.applyYamlUpdate()`
- 更新后触发相关的副作用（如成就检查）

### 10.4 数据路径

- 使用常量定义常用路径
- 路径字符串使用单引号
- 避免硬编码路径，使用配置或常量
