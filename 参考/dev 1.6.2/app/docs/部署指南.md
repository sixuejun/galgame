## 1. 概述

Eden System 是一个基于 Vue 3 的单页应用（SPA），专为 SillyTavern 环境设计。构建后会生成一个独立的 HTML 文件，包含所有必要的 JavaScript 和 CSS 代码。

**部署特点**：

- ✅ 单文件部署（`index.html`）
- ✅ 无需服务器，通过正则表达式注入到 SillyTavern
- ⚠️ 依赖 CDN 加载运行时库（Vue、Pinia、js-yaml、DOMPurify、marked）
- ⚠️ 需要网络连接以加载外部依赖

## 2. 环境要求

### 2.1 开发环境

| 工具    | 版本要求   | 说明                     |
| ------- | ---------- | ------------------------ |
| Node.js | >= 22.0.0  | JavaScript 运行时        |
| pnpm    | >= 10.0.0  | 包管理器（推荐 10.19.0） |
| Git     | 最新稳定版 | 版本控制（可选）         |

### 2.2 目标环境

| 组件            | 版本要求    | 说明                                                   |
| --------------- | ----------- | ------------------------------------------------------ |
| SillyTavern     | 最新稳定版  | AI 聊天界面                                            |
| JS-Slash-Runner | 最新版      | SillyTavern 插件，提供 JS 环境                         |
| 现代浏览器      | ES2022 支持 | Chrome/Edge/Firefox 最新版                             |
| 网络连接        | 必需        | 用于加载 CDN 依赖库                                    |
| st-chatu8       | 可选        | 图片生成服务网关插件（如需使用 AI 图片生成功能则必需） |

**关于 st-chatu8 插件**：

- **作用**: 图片生成服务网关，对接 Stable Diffusion、NovelAI、ComfyUI 等生图服务
- **是否必需**: 可选。如果不使用 AI 图片生成功能，可以不安装
- **安装位置**: SillyTavern 插件
- **配置方式**: 在设置页面选择图片生成服务为 `st-chatu8`

## 3. 构建流程

### 3.1 克隆项目

```bash
# 克隆仓库
git clone <repository-url>
cd SillyTavern-tongceng/app

# 或直接解压源代码压缩包
cd app
```

### 3.2 安装依赖

```bash
# 安装 pnpm（如果尚未安装）
npm install -g pnpm

# 安装项目依赖
pnpm install
```

### 3.3 构建生产版本

```bash
# 运行构建命令
pnpm run build
```

**构建过程**：

1. **类型检查**：`vue-tsc --noEmit` 检查 TypeScript 类型错误
2. **代码编译**：Vite 编译 Vue 组件和 TypeScript 代码
3. **代码压缩**：使用 Terser 压缩 JavaScript 代码
4. **CSS 处理**：提取并压缩 CSS 样式
5. **单文件打包**：使用 `vite-plugin-singlefile` 将所有资源内联到单个 HTML 文件

**构建输出**：

```
app/dist/
└── index.html    # 单文件应用（约 500KB - 1MB）
```

### 3.4 验证构建

```bash
# 检查构建产物
ls -lh dist/index.html

# 预览构建结果（可选）
pnpm run preview
```

## 4. 部署到 SillyTavern

### 4.1 配置正则表达式脚本

1. 打开 SillyTavern
2. 进入 **正则表达式（Regex）** 管理界面
3. 创建新的正则表达式脚本：

| 配置项   | 值                                             |
| -------- | ---------------------------------------------- |
| 脚本名称 | `伊甸园前端界面`                               |
| 查找正则 | `【\d+】`                                      |
| 替换内容 | ` ```html\n<dist/index.html 的完整内容>\n``` ` |
| 应用范围 | ✅ AI 输出 + ✅ 用户输入                       |

4. 保存并启用该脚本

### 4.2 复制构建产物

```bash
# 方法 1：手动复制
# 打开 app/dist/index.html，复制全部内容
# 粘贴到正则表达式脚本的"替换内容"中（记得用代码块包裹）

# 方法 2：使用命令行（Windows PowerShell）
Get-Content dist/index.html | Set-Clipboard

# 方法 3：使用命令行（Linux/macOS）
cat dist/index.html | pbcopy  # macOS
cat dist/index.html | xclip -selection clipboard  # Linux
```

### 4.3 激活系统

在角色卡的开场白或第一条消息中添加激活标记：`【00】`

## 5. 验证部署

### 5.1 检查清单

- [ ] JS-Slash-Runner 插件已安装并启用
- [ ] 正则表达式脚本已创建并启用
- [ ] `dist/index.html` 内容已正确复制到替换内容中
- [ ] 替换内容已用代码块（三个反引号）包裹
- [ ] 开场白中包含激活标记（如 `【00】`）

### 5.2 测试步骤

1. 在 SillyTavern 中创建新对话
2. 发送第一条消息（包含 `【00】` 标记）
3. 检查是否显示前端界面
4. 测试基本功能（导航、按钮点击等）

### 5.3 常见问题排查

| 问题               | 可能原因               | 解决方法                              |
| ------------------ | ---------------------- | ------------------------------------- |
| 界面未显示         | 正则脚本未启用         | 检查正则表达式脚本是否勾选"应用范围"  |
| 界面显示但无法交互 | JS-Slash-Runner 未安装 | 安装并启用 JS-Slash-Runner 插件       |
| 样式错乱           | HTML 内容未完整复制    | 重新复制 `dist/index.html` 的完整内容 |
| 控制台报错         | 外部依赖未加载         | 检查 SillyTavern 是否能访问 CDN 资源  |

## 6. 常见问题

### Q1: 构建失败怎么办？

**错误：类型检查失败**

```bash
# 运行类型检查查看详细错误
pnpm run type-check
```

**错误：依赖安装失败**

```bash
# 清除缓存并重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Q2: 如何更新部署？

1. 拉取最新代码或修改源代码
2. 重新运行 `pnpm run build`
3. 复制新的 `dist/index.html` 内容到正则表达式脚本
4. 刷新 SillyTavern 对话

### Q3: 如何自定义构建配置？

编辑 `app/vite.config.ts` 文件，修改构建选项：

```typescript
export default defineConfig({
  build: {
    target: 'es2022', // 目标浏览器版本
    minify: 'terser', // 压缩工具
    sourcemap: false, // 是否生成 source map
    cssMinify: true, // 是否压缩 CSS
  },
})
```

### Q4: 构建产物太大怎么办？

当前构建配置已经优化，但如果需要进一步减小体积：

1. 检查是否有未使用的依赖
2. 移除不必要的图片和资源
3. 考虑使用 CDN 加载外部依赖（需修改 `vite.config.ts`）
