## 1. 前端界面问题

### 1.1 界面未显示

**症状**：对话中没有显示前端界面，只有纯文本

**可能原因**：

1. JS-Slash-Runner 插件未安装或未启用
2. 正则表达式脚本配置错误
3. 开场白中缺少激活标记

**排查步骤**：

```bash
# 步骤 1：检查 JS-Slash-Runner 插件
1. 打开 SillyTavern 扩展页面
2. 确认 JS-Slash-Runner 已安装并启用
3. 如果未安装，搜索并安装该插件
4. 重启 SillyTavern

# 步骤 2：检查正则表达式脚本
1. 打开正则表达式管理界面
2. 确认"伊甸园前端界面"脚本已启用
3. 检查"应用范围"是否勾选了"AI 输出"和"用户输入"
4. 确认替换内容已用代码块（三个反引号）包裹

# 步骤 3：检查激活标记
1. 在开场白或第一条消息中添加 【00】
2. 重新开始对话
```

**解决方法**：

- 确保 `app/dist/index.html` 的内容已正确复制到正则表达式脚本
- 检查替换内容格式：` ```html\n<完整的 index.html 内容>\n``` `

### 1.2 界面显示但无法交互

**症状**：界面显示正常，但按钮点击无响应

**可能原因**：

1. JavaScript 执行被阻止
2. CDN 依赖（Vue、Pinia、js-yaml、DOMPurify、marked）未加载
3. 网络连接问题导致 CDN 资源加载失败
4. 浏览器控制台有错误

**排查步骤**：

```javascript
// 步骤 1：打开浏览器控制台（F12）
// 查看是否有错误信息或网络请求失败

// 步骤 2：检查 CDN 依赖是否加载成功
console.log(window.Vue) // 应该输出 Vue 对象
console.log(window.Pinia) // 应该输出 Pinia 对象
console.log(window.jsyaml) // 应该输出 js-yaml 对象
console.log(window.DOMPurify) // 应该输出 DOMPurify 对象
console.log(window.marked) // 应该输出 marked 对象

// 步骤 3：检查应用是否挂载
console.log(document.querySelector('#app')) // 应该输出 DOM 元素

// 步骤 4：检查网络请求
// 打开浏览器控制台 -> Network 标签
// 查看是否有 unpkg.com 的请求失败
```

**解决方法**：

- 确保网络连接正常，能够访问 unpkg.com CDN
- 检查浏览器是否阻止了第三方脚本加载
- 清除浏览器缓存并刷新页面
- 如果 CDN 访问受限，考虑使用代理或 VPN

### 1.3 样式错乱

**症状**：界面显示但样式混乱、布局错误

**可能原因**：

1. CSS 未正确内联
2. HTML 内容被截断
3. 浏览器兼容性问题

**排查步骤**：

```bash
# 步骤 1：检查 HTML 完整性
1. 打开正则表达式脚本
2. 检查替换内容是否完整（应该包含 <!DOCTYPE html> 到 </html>）
3. 确认没有被截断

# 步骤 2：检查浏览器兼容性
1. 使用 Chrome/Edge/Firefox 最新版
2. 确保浏览器支持 ES2022 语法
```

**解决方法**：

- 重新复制 `app/dist/index.html` 的完整内容
- 使用现代浏览器（Chrome 90+、Edge 90+、Firefox 88+）

## 2. AI 通信问题

### 2.1 AI 响应格式错误

**症状**：控制台显示"YAML 解析失败"或"数据格式错误"

**可能原因**：

1. AI 返回的 YAML 格式不正确
2. 系统提示词未正确配置
3. AI 模型理解能力不足

**排查步骤**：

```javascript
// 步骤 1：开启调试模式查看 AI 响应
localStorage.setItem('eden-system-settings', JSON.stringify({ debugMode: true }))

// 步骤 2：查看控制台日志
// 找到 "AI 响应原始内容" 日志，检查 YAML 格式

// 步骤 3：手动验证 YAML
// 复制 AI 响应到 YAML 验证器（如 yamllint.com）
```

**解决方法**：

1. 确保世界书中的系统提示词已正确配置
2. 添加附属提示词"YAML格式规范"强化格式要求
3. 调整 AI 温度参数（降低温度可提高格式稳定性）
4. 使用更强大的 AI 模型（如 GPT-4）

### 2.2 AI 请求超时

**症状**：长时间等待后显示"请求超时"

**可能原因**：

1. 网络连接问题
2. AI 服务响应慢
3. 请求被限流

**排查步骤**：

```javascript
// 步骤 1：检查网络连接
// 打开浏览器控制台 -> Network 标签
// 查看是否有失败的请求

// 步骤 2：检查重试设置
const settings = JSON.parse(localStorage.getItem('eden-system-settings'))
console.log('最大重试次数:', settings.maxRetries)
console.log('重试延迟:', settings.retryDelay)
```

**解决方法**：

- 检查网络连接，确保能访问 AI 服务
- 在设置页面增加最大重试次数
- 增加重试延迟时间
- 联系 AI 服务提供商检查服务状态

### 2.3 图片生成失败（st-chatu8）

**症状**：显示"图片生成失败"或"ST-ChatU8 服务不可用"

**可能原因**：

1. st-chatu8 插件未安装或未启用
2. st-chatu8 服务未正确配置
3. 图片生成超时
4. 提示词格式错误

**排查步骤**：

```javascript
// 步骤 1：检查 st-chatu8 API 是否可用
console.log('eventOn:', typeof window.eventOn)
console.log('eventEmit:', typeof window.eventEmit)
console.log('eventRemoveListener:', typeof window.eventRemoveListener)

// 步骤 2：检查图片生成服务设置
const settings = JSON.parse(localStorage.getItem('eden-system-settings'))
console.log('图片生成服务:', settings.imageGenerationService)
console.log('超时时间:', settings.stChatu8ImageTimeout)

// 步骤 3：开启调试模式查看详细日志
localStorage.setItem(
  'eden-system-settings',
  JSON.stringify({
    ...settings,
    debugMode: true,
  })
)
```

**解决方法**：

- 确保 st-chatu8 插件已安装并启用（位于外部项目 `st-chatu8-main`）
- 检查 st-chatu8 的配置（Stable Diffusion、NovelAI、ComfyUI 等）
- 在设置页面增加图片生成超时时间（默认 30 秒）
- 如果不使用图片生成功能，可以在设置中切换到 `pollinations` 服务
- 确保提示词为英文（st-chatu8 通常需要英文提示词）

## 3. 数据加载问题

### 3.1 世界书数据加载失败

**症状**：显示"数据加载失败"或"世界书连接失败"

**可能原因**：

1. 世界书未绑定到角色卡
2. 世界书条目未启用
3. 数据格式错误

**排查步骤**：

```bash
# 步骤 1：检查世界书绑定
1. 打开角色卡设置
2. 确认世界书已设置为"主世界书"
3. 确认世界书条目已启用

# 步骤 2：检查数据格式
1. 打开世界书条目
2. 确认数据是有效的 YAML 格式
3. 使用 YAML 验证器检查语法
```

**解决方法**：

- 重新绑定世界书到角色卡
- 检查并修复 YAML 语法错误
- 使用示例数据（`public/data.yaml`）作为参考

### 3.2 存档加载失败

**症状**：点击"加载存档"后无响应或显示错误

**可能原因**：

1. 存档数据损坏
2. 存档格式不兼容
3. localStorage 空间不足

**排查步骤**：

```javascript
// 步骤 1：检查 localStorage
console.log('存储空间使用:', JSON.stringify(localStorage).length, '字节')

// 步骤 2：查看存档列表
const saves = JSON.parse(localStorage.getItem('eden-system-saves') || '[]')
console.log('存档数量:', saves.length)
console.log('存档列表:', saves)

// 步骤 3：验证存档数据
saves.forEach((save, index) => {
  try {
    JSON.parse(save.data)
    console.log(`存档 ${index} 有效`)
  } catch (error) {
    console.error(`存档 ${index} 损坏:`, error)
  }
})
```

**解决方法**：

- 删除损坏的存档
- 清理不必要的存档释放空间
- 导出重要存档到文件备份

## 4. 性能问题

### 4.1 界面卡顿

**症状**：界面响应慢、滚动不流畅

**可能原因**：

1. 数据量过大
2. 浏览器性能不足
3. 内存泄漏

**排查步骤**：

```javascript
// 步骤 1：检查数据大小
const gameData = JSON.stringify(window.__PINIA_STORES__?.game?.gameData || {})
console.log('游戏数据大小:', gameData.length, '字节')

// 步骤 2：检查内存使用
// 打开浏览器控制台 -> Performance 标签
// 录制性能分析，查看内存使用情况
```

**解决方法**：

- 清理不必要的游戏数据
- 关闭其他浏览器标签页
- 使用性能更好的浏览器（推荐 Chrome/Edge）

## 5. 调试工具

### 5.1 开启调试模式

```javascript
// 方法 1：在设置页面开启调试模式

// 方法 2：在控制台执行
localStorage.setItem(
  'eden-system-settings',
  JSON.stringify({
    debugMode: true,
    maxRetries: 3,
    maxDataRetries: 3,
    retryDelay: 1000,
    autoSave: true,
    imageGenerationService: 'pollinations',
    stChatu8ImageTimeout: 30000,
  })
)

// 刷新页面使设置生效
location.reload()
```

### 5.2 查看游戏数据

```javascript
// 获取 Pinia stores
const gameStore = window.__PINIA_STORES__?.game
const navigationStore = window.__PINIA_STORES__?.navigation
const settingsStore = window.__PINIA_STORES__?.settings
```
