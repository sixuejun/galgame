# 测试指南

## 概述

本指南介绍如何在 Eden System 项目中编写和运行测试。项目使用 Vitest 作为测试框架，配合 Vue Test Utils 进行组件测试。

## 测试框架和工具

### 核心工具

- **Vitest 4.0.3**: 基于 Vite 的快速测试框架
- **@vue/test-utils 2.4.6**: Vue 3 官方测试工具库
- **happy-dom 20.0.8**: 轻量级 DOM 环境
- **@vitest/coverage-v8**: 代码覆盖率工具
- **@vitest/ui**: 可视化测试界面

### 测试命令

```bash
# 监听模式（开发时使用）
pnpm run test

# 单次运行（CI 环境）
pnpm run test:run

# UI 模式（可视化测试结果）
pnpm run test:ui

# 测试覆盖率
pnpm run test:coverage
```

## 组件测试

### 基本结构

```typescript
import { describe, it, expect, beforeEach, vi } from 'vitest'
import { mount } from '@vue/test-utils'
import { createPinia, setActivePinia } from 'pinia'
import ComponentName from '../ComponentName.vue'

describe('ComponentName', () => {
  beforeEach(() => {
    // 每个测试前重置状态
    setActivePinia(createPinia())
    vi.clearAllMocks()
  })

  describe('渲染测试', () => {
    it('应该渲染组件', () => {
      const wrapper = mount(ComponentName)
      expect(wrapper.exists()).toBe(true)
    })
  })

  describe('Props 测试', () => {
    it('应该正确接收和显示 props', () => {
      const wrapper = mount(ComponentName, {
        props: { title: '测试标题' },
      })
      expect(wrapper.text()).toContain('测试标题')
    })
  })

  describe('事件测试', () => {
    it('应该触发事件', async () => {
      const wrapper = mount(ComponentName)
      await wrapper.find('button').trigger('click')
      expect(wrapper.emitted('click')).toBeTruthy()
    })
  })
})
```

## Mock 使用

### Mock 子组件

当测试父组件时，通常需要 Mock 子组件以隔离测试：

```typescript
// Mock 子组件
vi.mock('../ChildComponent.vue', () => ({
  default: {
    name: 'ChildComponent',
    template: '<div class="child-mock"></div>',
    props: ['title', 'count'],
    emits: ['update', 'delete'],
  },
}))

describe('ParentComponent', () => {
  it('应该渲染 Mock 的子组件', () => {
    const wrapper = mount(ParentComponent)
    expect(wrapper.find('.child-mock').exists()).toBe(true)
  })
})
```

## 测试最佳实践

### 1. 测试独立性

每个测试应该独立运行，不依赖其他测试的状态：

```typescript
describe('Component', () => {
  beforeEach(() => {
    // 每个测试前重置状态
    setActivePinia(createPinia())
    vi.clearAllMocks()
    localStorage.clear()
  })

  afterEach(() => {
    // 每个测试后清理
    vi.restoreAllMocks()
  })
})
```

### 2. 使用描述性的测试名称

```typescript
// ❌ 不好
it('test 1', () => {})

// ✅ 好
it('应该在用户点击按钮时显示成功消息', () => {})
```

### 3. 遵循 AAA 模式

Arrange（准备）、Act（执行）、Assert（断言）：

```typescript
it('应该正确计算总价', () => {
  // Arrange - 准备测试数据
  const items = [
    { price: 10, quantity: 2 },
    { price: 5, quantity: 3 },
  ]

  // Act - 执行被测试的操作
  const total = calculateTotal(items)

  // Assert - 验证结果
  expect(total).toBe(35)
})
```

### 4. 测试边界情况

```typescript
describe('validateAge', () => {
  it('应该接受有效年龄', () => {
    expect(validateAge(25)).toBe(true)
  })

  it('应该拒绝负数', () => {
    expect(validateAge(-1)).toBe(false)
  })

  it('应该拒绝零', () => {
    expect(validateAge(0)).toBe(false)
  })

  it('应该拒绝过大的数字', () => {
    expect(validateAge(200)).toBe(false)
  })

  it('应该处理边界值', () => {
    expect(validateAge(1)).toBe(true)
    expect(validateAge(150)).toBe(true)
  })
})
```

### 5. 避免测试实现细节

```typescript
// ❌ 不好 - 测试实现细节
it('应该调用内部方法 _processData', () => {
  const spy = vi.spyOn(component, '_processData')
  component.handleClick()
  expect(spy).toHaveBeenCalled()
})

// ✅ 好 - 测试行为和结果
it('应该在点击后显示处理后的数据', () => {
  component.handleClick()
  expect(component.displayData).toBe('processed data')
})
```

### 6. 使用 Mock 隔离依赖

```typescript
// ✅ 好 - 隔离外部依赖
vi.mock('../../services/api', () => ({
  fetchData: vi.fn(() => Promise.resolve({ data: 'test' })),
}))

it('应该显示获取的数据', async () => {
  const wrapper = mount(Component)
  await flushPromises()
  expect(wrapper.text()).toContain('test')
})
```

### 7. 测试异步代码

```typescript
import { flushPromises } from '@vue/test-utils'

it('应该处理异步操作', async () => {
  const wrapper = mount(Component)

  // 触发异步操作
  await wrapper.find('button').trigger('click')

  // 等待所有 Promise 完成
  await flushPromises()

  // 验证结果
  expect(wrapper.text()).toContain('加载完成')
})
```

### 8. 测试错误处理

```typescript
it('应该显示错误消息', async () => {
  // Mock 失败的 API 调用
  vi.mocked(APIService.fetch).mockRejectedValue(new Error('Network error'))

  const wrapper = mount(Component)
  await wrapper.find('button').trigger('click')
  await flushPromises()

  expect(wrapper.find('.error-message').text()).toContain('Network error')
})
```

### 9. 测试可访问性

```typescript
it('应该有正确的 ARIA 属性', () => {
  const wrapper = mount(Component)

  expect(wrapper.attributes('role')).toBe('button')
  expect(wrapper.attributes('aria-label')).toBe('关闭对话框')
  expect(wrapper.attributes('aria-pressed')).toBe('false')

```

### 10. 保持测试简洁

```typescript
// ❌ 不好 - 测试太多内容
it('应该处理所有用户交互', () => {
  // 测试点击
  // 测试输入
  // 测试提交
  // 测试验证
  // ... 太多了
})

// ✅ 好 - 每个测试只测试一件事
it('应该在点击时切换状态', () => {
  // 只测试点击行为
})

it('应该验证输入格式', () => {
  // 只测试验证逻辑
})
```

## 总结

编写高质量的测试需要：

1. ✅ **保持测试独立**：每个测试应该独立运行
2. ✅ **使用描述性名称**：清楚地说明测试的目的
3. ✅ **遵循 AAA 模式**：准备、执行、断言
4. ✅ **测试行为而非实现**：关注用户可见的行为
5. ✅ **使用 Mock 隔离依赖**：避免测试外部系统
6. ✅ **测试边界情况**：包括正常、异常和边界情况
7. ✅ **保持测试简洁**：每个测试只测试一件事
8. ✅ **追求合理的覆盖率**：目标 80% 以上

通过遵循这些最佳实践，你可以编写出可靠、可维护的测试代码，提高项目的整体质量。
