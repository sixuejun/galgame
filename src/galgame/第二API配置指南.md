# 第二 API 配置指南

## 📋 概述

第二 API 是一个独立的 API 配置，用于执行特定任务（如弹幕生成、商店刷新、系统对话、猜谜等），而不影响主 API 的使用。这样可以：

- **分离任务**：主 API 用于剧情生成，第二 API 用于辅助功能
- **成本优化**：可以为辅助功能使用更便宜的模型
- **性能优化**：避免辅助功能影响主剧情生成
- **灵活配置**：可以使用不同的提示词预设

---

## 🎯 核心特性

### 1. 独立 API 配置

第二 API 使用完全独立的配置：
- **接口地址**：独立的 API URL
- **API 密钥**：独立的密钥
- **模型选择**：可以选择不同的模型
- **预设选择**：可以使用不同的提示词预设

### 2. 预设支持

第二 API 支持使用酒馆的预设（Preset）：
- 选择预设后，会使用该预设的提示词等配置
- 但 API 请求仍使用第二 API 的地址和密钥
- 请求完成后会自动恢复原预设

**工作流程**：
```
1. 保存当前预设名称
2. 加载指定的第二 API 预设
3. 使用第二 API 的 custom_api 配置发送请求
4. 请求完成后恢复原预设
```

### 3. 重试机制

第二 API 具有自动重试机制：
- 请求失败后会自动重试最多 2 次
- 重试间隔逐渐增加
- 所有重试失败后才会提示错误

## ⚙️ 配置步骤

### 1. 打开设置面板

在游戏界面中，点击设置按钮打开设置面板。

### 2. 找到"第二 API"配置区域

在设置面板中找到"API 配置"部分，其中有"第二 API"配置区域。

### 3. 填写基础配置

**接口地址**：
```
https://api.openai.com/v1
```
或其他兼容 OpenAI API 格式的接口地址。

**API 密钥**：
```
sk-...
```
你的 API 密钥。

**模型名称**：
- 可以手动输入模型名称
- 或点击"拉取模型"按钮自动获取可用模型列表

**使用的预设**：
- 选择"不使用预设"：使用默认参数
- 选择具体预设：使用该预设的提示词配置
- 点击"刷新"按钮更新预设列表

### 4. 测试连接

点击"测试连接"按钮，验证配置是否正确：
- ✓ 连接成功：配置正确，可以使用
- ✗ 连接失败：检查接口地址和密钥是否正确

### 5. 高级参数（可选）

**温度 (Temperature)**：
- 范围：0-2
- 默认：使用预设值
- 说明：控制输出的随机性

**最大 Token 数**：
- 范围：1-无限
- 默认：使用预设值
- 说明：限制单次回复的最大长度

**Top P**：
- 范围：0-1
- 默认：使用预设值
- 说明：核采样参数

**Top K**：
- 范围：0-200
- 默认：使用预设值
- 说明：Top-K 采样参数

## 🔧 使用场景

### 1. 弹幕生成

当启用弹幕功能时，第二 API 会在每次 AI 回复后自动生成弹幕：

```typescript
// 系统会自动调用
await store.callSecondApi('danmaku', {
  contentText: '当前消息内容',
  chatHistory: [...] // 可选，如果启用了"发送时附带聊天记录"
});
```

### 2. 商店刷新

在商店模块中，点击"刷新商店"会使用第二 API 生成商品列表：

```typescript
await store.callSecondApi('shop', {
  systemPrompt: '生成商品列表的提示词'
});
```

### 3. 系统对话

在系统联系人对话中，使用第二 API 进行对话：

```typescript
await store.callSecondApi('system', {
  ordered_prompts: [
    { role: 'system', content: '系统提示词' },
    { role: 'user', content: '用户消息' }
  ]
});
```

### 4. 猜谜游戏

在猜谜模块中，使用第二 API 进行猜谜对话：

```typescript
await store.callSecondApi('riddle', {
  ordered_prompts: [...]
});
```

### 5. 总结生成

自动生成对话总结：

```typescript
await store.callSecondApi('summary', {
  systemPrompt: '总结提示词'
});
```

### 6. 生图标签生成

为生图功能生成标签：

```typescript
await store.callSecondApi('imageTag', {
  systemPrompt: '生成图片标签的提示词'
});
```

## 📊 API 任务分配

可以为每个任务指定使用哪个 API：

- **主 API**：使用酒馆主界面配置的 API
- **第二 API**：使用第二 API 配置
- **禁用**：不执行该任务

**配置位置**：设置面板 → API 配置 → API 任务分配

**可配置的任务**：
- 弹幕生成
- 总结生成
- 生图标签生成
- 变量更新

## 🐛 故障排除

### 问题 1：连接失败

**可能原因**：
- 接口地址错误
- API 密钥错误
- 网络问题

**解决方法**：
1. 检查接口地址是否正确（注意末尾不要有多余的斜杠）
2. 检查 API 密钥是否正确
3. 尝试在浏览器中直接访问接口地址

### 问题 2：模型列表获取失败

**可能原因**：
- API 不支持 `/models` 接口
- 密钥权限不足

**解决方法**：
1. 手动输入模型名称
2. 检查 API 文档确认是否支持模型列表接口

### 问题 3：请求失败

**可能原因**：
- 网络问题
- API 超时
- 模型不可用

**解决方法**：
1. 检查网络连接
2. 检查 API 状态
3. 尝试更换模型
4. 系统会自动重试 2 次

### 问题 4：预设加载失败

**可能原因**：
- 预设不存在
- 预设名称错误

**解决方法**：
1. 点击"刷新"按钮更新预设列表
2. 检查预设是否存在于酒馆中
3. 选择"不使用预设"

## 💡 最佳实践

### 1. 成本优化

- **主 API**：使用高质量模型（如 GPT-4）用于剧情生成
- **第二 API**：使用便宜模型（如 GPT-3.5）用于辅助功能

### 2. 性能优化

- 为第二 API 设置较低的 `max_tokens`，避免生成过长内容
- 使用合适的温度参数，平衡创意和稳定性

### 3. 预设管理

- 为第二 API 创建专门的预设
- 预设中包含针对辅助功能优化的提示词
- 定期更新预设以优化效果

### 4. 监控状态

- 定期检查第二 API 状态
- 如果出现降级，及时处理
- 查看生成历史，了解使用情况

---

## 🔐 安全建议

1. **不要共享 API 密钥**：API 密钥是敏感信息，不要分享给他人
2. **使用独立密钥**：为第二 API 使用独立的密钥，便于管理和撤销
3. **设置使用限制**：在 API 提供商处设置使用限制，避免超额消费
4. **定期轮换密钥**：定期更换 API 密钥，提高安全性
