# galgame 开发与实时修改说明

## 为什么看不到实时修改？

实时修改依赖**三件事同时满足**：

1. **本地有静态资源服务在 5500 端口**  
   角色卡里脚本加载的是 `http://localhost:5500/dist/galgame/index.js`，浏览器必须能从该 URL 拿到最新打包结果。

2. **源码变更后自动打包**  
   需要运行 `pnpm watch`，这样修改 `src/galgame` 后会自动编译到 `dist/galgame/`。

3. **酒馆助手已开启实时监听**  
   在酒馆设置里找到「酒馆助手」→「实时监听」→ 勾选「允许监听」。这样 webpack 编译完成后会通过 6621 端口的 socket 通知酒馆刷新脚本/界面。

## 推荐操作步骤

### 方式一：用本项目的静态服务（推荐）

1. **终端 1**：在项目根目录执行  

   ```bash
   pnpm run dev:serve
   ```

   会在 **5500** 端口启动静态服务（`http-server`），自动加上 CORS 响应头（`Access-Control-Allow-Origin: *`）并禁用缓存，因此 `http://localhost:5500/dist/galgame/index.js` 可被酒馆页面跨域加载并及时拿到新版本。  
   如果提示 5500 端口被占用，请先关闭占用 5500 的旧服务（常见是 Live Server）。

2. **终端 2**：在项目根目录执行  

   ```bash
   pnpm watch
   ```

   监听源码并自动打包到 `dist/`，编译完成后会通过 6621 通知酒馆。

3. 在酒馆中：确保「酒馆助手 → 实时监听 → 允许监听」已开启。

4. 用浏览器打开酒馆页面（例如 `http://127.0.0.1:8000/` 或你在 launch.json 里配置的地址），使用已配置「流式楼层界面-实时修改」脚本的角色卡聊天。  
   修改 `src/galgame` 下代码并保存后，等 watch 编译完成，酒馆会收到通知并热重载脚本，即可看到实时修改。

### 方式二：用 VS Code Live Server

1. 用 VS Code 打开项目根目录，对根目录或任意文件右键 → **Open with Live Server**（需已安装 Live Server 插件）。  
   默认会在 **5500** 端口提供当前工作区，保证 `http://localhost:5500/dist/galgame/index.js` 可访问。

2. 同样在项目根目录运行 `pnpm watch`。

3. 酒馆中开启「酒馆助手 → 实时监听 → 允许监听」。

## 端口说明

| 端口 | 用途                                                                                                                     |
| ---- | ------------------------------------------------------------------------------------------------------------------------ |
| 5500 | 静态资源：浏览器从这里加载 `dist/galgame/index.js` 等。由 `pnpm run dev:serve` 或 Live Server 提供。                     |
| 6621 | 酒馆助手监听：webpack watch 编译完成后向此端口发通知，酒馆收到后热重载。在 `webpack.config.ts` 的 `config.port` 中配置。 |
| 8000 | 你的酒馆页面地址（以 `.vscode/launch.json` 或实际部署为准）。                                                            |

## 脚本地址请使用 5500

角色卡中「流式楼层界面-实时修改」的脚本地址必须为：

- **`http://localhost:5500/dist/galgame/index.js`**

若曾改为其他端口（例如 5510），请改回 5500，否则会出现 `net::ERR_CONNECTION_REFUSED` 或无法加载脚本。

## 仍然看不到更新时请检查

- 是否同时开了 **dev:serve（或 Live Server）** 和 **pnpm watch**。
- 若使用本项目的 `dev:serve`，请确认命令是最新脚本（含 `http-server --cors`），否则会出现 `No 'Access-Control-Allow-Origin' header`（酒馆在 127.0.0.1:8000 时跨域会拦截脚本）。
- 角色卡里脚本地址是否为 **`http://localhost:5500/dist/galgame/index.js`**（端口 5500）。
- 酒馆是否打开了「酒馆助手 → 实时监听 → 允许监听」，且浏览器没有禁用该页面的 WebSocket。
- 若浏览器强缓存了脚本，可尝试硬刷新（Ctrl+Shift+R）或打开开发者工具 → Network 勾选 Disable cache 后刷新。
