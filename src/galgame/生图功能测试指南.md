# 生图功能测试指南

## 前置条件

1. 安装支持生图事件的前端助手插件
2. 配置第二API（用于生成图片tag）
3. 确保酒馆助手正常运行

## 测试步骤

### 1. 基础配置测试

#### 1.1 仅背景模式
1. 打开设置面板
2. 开启"生图总开关"
3. 仅开启"背景生成"，关闭"CG生成"
4. 发送消息触发AI回复
5. **预期结果**：
   - 生成背景图片并显示在舞台上
   - 不显示扇形卡牌队列
   - Console显示：`[ImageGen] Requesting background image`

#### 1.2 仅CG模式
1. 关闭"背景生成"，开启"CG生成"
2. 发送消息触发AI回复
3. **预期结果**：
   - 生成CG图片并显示在舞台上
   - 不显示扇形卡牌队列
   - Console显示：`[ImageGen] Requesting CG image`

### 2. 扇形卡牌队列测试

#### 2.1 开启卡牌队列
1. 同时开启"背景生成"和"CG生成"
2. 发送多条消息（至少3条）
3. **预期结果**：
   - 左侧中间位置出现叠卡角
   - 显示队列数量徽章
   - 悬停时展开为扇形轮盘

#### 2.2 卡牌交互
1. 悬停在卡牌队列上
2. 点击不同的卡牌
3. **预期结果**：
   - 卡牌展开为扇形
   - 点击后舞台切换到对应图片
   - 卡牌有BG/CG标记

#### 2.3 队列管理
1. 生成超过10张图片
2. 打开设置面板查看队列数量
3. 点击"清空队列"按钮
4. **预期结果**：
   - 最多保留10张图片
   - 清空后卡牌队列消失

### 3. 插入位置判断测试

#### 3.1 CG场景块检测
发送包含以下内容的消息：
```
[[character||角色名：程北极||CG场景：告白||台词：我喜欢你]]
```
**预期结果**：生成CG图片

#### 3.2 场景变化检测
发送包含以下内容的消息：
```
[[character||角色名：程北极||场景：学校||台词：早上好]]
```
**预期结果**：生成背景图片

#### 3.3 两者都存在
发送包含以下内容的消息：
```
[[character||角色名：程北极||场景：家中||台词：欢迎]]
[[character||角色名：程北极||CG场景：拥抱||台词：抱紧你]]
```
**预期结果**：根据优先级设置生成对应图片

### 4. 第二API生图tag测试

#### 4.1 配置第二API
1. 在设置面板配置第二API
2. 在"API任务分配"中设置"生图Tag"为"第二API"
3. 发送消息触发AI回复

#### 4.2 检查生成结果
1. 打开浏览器Console
2. 查看日志：`[ImageGen] Generated tags via second API: xxx`
3. 检查消息末尾是否有HTML注释：
   ```html
   <!-- 第二API生图tag: post-apocalyptic, wasteland, newspaper style -->
   ```

### 5. 优先级设置测试

#### 5.1 CG优先
1. 设置"两者都存在时的优先级"为"CG优先"
2. 发送同时包含场景和CG场景的消息
3. **预期结果**：生成CG图片

#### 5.2 背景优先
1. 设置优先级为"背景优先"
2. 发送同样的消息
3. **预期结果**：生成背景图片

## 常见问题排查

### 问题1：生图请求未发送
- 检查生图总开关是否开启
- 检查对应的背景/CG开关是否开启
- 查看Console是否有错误

### 问题2：第二API生图tag未生成
- 确认第二API已配置
- 检查"生图Tag"设置为"第二API"
- 测试第二API连接是否正常

### 问题3：卡牌队列不显示
- 确认同时开启了背景和CG
- 检查是否有图片生成成功
- 查看imageCardQueue状态

### 问题4：图片类型判断错误
- 检查消息格式是否正确
- 查看Console中的判断日志
- 调整优先级设置

## 调试技巧

### Console日志
关键日志标记：
- `[ImageGen]`：生图相关日志
- `Requesting background image`：请求背景
- `Requesting CG image`：请求CG
- `Generated tags via second API`：第二API生成tag
- `Both detected, using xxx priority`：优先级判断

### 状态检查
在Console中执行：
```javascript
// 查看卡牌队列
window.__galgameState.mainStore.imageCardQueue

// 查看当前舞台图片
window.__galgameState.mainStore.stageBackgroundImage
window.__galgameState.mainStore.stageCgImage

// 查看设置
window.__galgameState.mainStore.settings
```

## 性能注意事项

1. **队列限制**：最多10张图片，避免内存占用过大
2. **图片格式**：建议使用base64格式，大小控制在2MB以内
3. **生成频率**：避免短时间内大量生图请求

## 已知限制

1. 不支持同时生成背景和CG（需要选择优先级）
2. 卡牌队列仅在同时开启两个开关时显示
3. 图片类型判断依赖消息格式，需要正确的标记
