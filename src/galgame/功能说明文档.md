## 一、整体架构

### 技术栈

- **Vue 3** + **Pinia** 状态管理
- **Tailwind CSS v4** + 自定义 CSS 变量（`theme.css`）
- **流式楼层界面**：基于 `mountStreamingMessages` 挂载，每个 AI 消息楼层独立 iframe Vue 实例
- 持久化：设置存于 `script` 变量，游戏数据存于 `chat` 变量

### 视觉风格

- 复古做旧报纸质感：泛黄纸张、褪色油墨、折痕污渍
- 色调：灰暗土黄 `#d4c5a0`、灰黑 `#2a2420`、锈色 `#8b4513`
- 16:9 固定比例视口，报纸装饰元素（EXTRA EDITION / 末日旧闻 / 第壹百零七期）

### 图层结构（从下到上）

1. **背景层**：渐变 + 报纸标题水印
2. **生图背景层**：第二API生成的背景图片
3. **生图CG层**：第二API生成的CG图片
4. **弹幕层**：screen-space，不受场景抖动影响
5. **暗角层**：径向渐变暗角
6. **噪点层**：SVG fractalNoise 纹理
7. **扇形卡牌队列**（z-index 30）：左侧中间位置，悬停展开
8. **UI 层**（z-index 20）：快捷菜单 + 对话框
9. **选项层**：玩家选择面板
10. **覆盖面板**（z-index 50）：设置/历史/角色/玩法
11. **模块面板**（z-index 55）：各玩法模块 overlay
12. **系统通知**（z-index 9999）：Toast 弹窗，永远在最上层

## 四、消息解析系统

### 4.1 内容提取规则

VN 主舞台**仅解析 `<content>` 标签内的内容**，标签外的内容全部忽略。

若消息中**没有 `<content>` 标签**，VN 主舞台不显示任何内容（静默忽略）。

### 4.2 世界书资源定义格式

在世界书中创建条目，内容为 JSON 格式：

#### 背景资源

```json
{
  "type": "background",
  "backgrounds": [
    {
      "name": "废墟",
      "file": "https://example.com/bg_ruins.jpg",
      "textMappings": ["废墟", "荒地", "废弃建筑", "破败的街道"]
    }
  ]
}
```

#### CG资源

```json
{
  "type": "cg",
  "cgs": [
    {
      "name": "拥抱",
      "file": "https://example.com/cg_hug.jpg",
      "textMappings": ["拥抱", "抱住", "紧紧相拥"]
    }
  ]
}
```

#### 立绘资源

```json
{
  "type": "sprite",
  "sprites": [
    {
      "name": "程北极",
      "file": "https://example.com/sprite_cheng.png",
      "textMappings": ["程北极", "北极", "小程"]
    }
  ]
}
```

### 4.3 消息格式说明

`<content>` 标签内的内容使用以下格式：

#### 角色对话

```
[[character||角色名：程北极||场景：废墟||动作：抱胸||表情：生气||台词：你怎么这样！]]
```

#### CG场景

```
[[character||角色名：程北极||CG场景：拥抱||台词：终于找到你了...]]
```

#### 旁白

```
[[narration||场景：废墟||旁白：天色渐暗，废墟中传来奇怪的声音。]]
```

#### 黑屏文字

```
[[blacktext||黑屏文字：三天后...]]
```

#### 用户消息

```
[[user||场景：废墟||台词：我们快走吧！]]
```

### 4.4 场景继承机制

如果某个消息块没有指定场景，会自动继承上一个消息块的场景。

### 4.5 模糊匹配规则

系统会对场景名、角色名进行模糊匹配：

1. **去除特殊符号**：`-`、`/`、`_`、`.`、空格等
2. **转为小写**
3. **包含匹配**：只要包含关键词即可

### 4.6 分层结构

系统使用以下分层结构（从上到下）：

1. **UI层** (z-index: 20) - 对话框、菜单等UI元素
2. **弹幕层** (z-index: 5) - 弹幕显示
3. **CG层** (z-index: 4) - CG图片和生成的图片
4. **效果层** (z-index: 3) - 暗角、噪点等视觉效果
5. **立绘层** (z-index: 2) - 角色立绘
6. **背景层** (z-index: 1) - 背景图片

## 六、生图系统与扇形卡牌队列

### 6.1 概述

生图系统通过前端助手事件与外部插件通信，支持在主API回复后自动调用第二API生成图片tag，并通过插件生成背景图或CG图。

### 6.2 生图开关逻辑

| 开关状态 | 行为 |
|---------|------|
| 仅背景 | 只生成背景图片，默认生成的图片作为背景 |
| 仅CG | 只生成CG图片，默认生成的图片作为CG |
| 两者都开 | 根据消息内容判断生成类型，开启扇形卡牌队列 |
| 都关闭 | 不调用生图 |

### 6.3 插入位置判断规则

当同时开启背景和CG生图时，系统会根据消息内容判断：

1. **优先判定CG**：如果消息中出现CG场景块
2. **判定背景**：如果出现场景变化（但不是CG场景）
3. **两者都存在**：使用用户设置的优先级（CG优先/背景优先）

### 6.4 第二API生图流程

1. **主API回复完成**：触发 `GENERATION_ENDED` 事件
2. **判断生图类型**：根据消息内容和开关状态判断
3. **调用第二API**：生成图片tag（如果启用）
4. **发送生图请求**：通过 `eventEmit` 发送到外部插件
5. **写入消息末尾**：将第二API生成的tag作为HTML注释写入

### 6.5 扇形卡牌队列 (Image Deck)

#### 显示条件
- 同时打开背景和CG生图开关
- 至少有1张图片在队列中

#### 交互方式
- **默认收起**：左侧中间位置显示叠卡角，显示队列数量
- **悬停展开**：变成扇形轮盘，按顺序排列图片
- **点击卡牌**：立即切换舞台显示为该卡牌对应图片

#### 队列管理
- 最多保存10张图片
- 超过10张时自动移除最旧的
- 可在设置面板中清空队列

#### 卡牌样式
- **背景卡牌**：灰褐色边框，标记"BG"
- **CG卡牌**：锈色边框，标记"CG"
- 悬停时放大并高亮边框

### 6.6 配置选项

在设置面板的"生图"部分：

- **生图总开关**：控制整个生图功能
- **背景生成**：开启背景图生成
- **CG生成**：开启CG图生成
- **两者都存在时的优先级**：CG优先 / 背景优先
- **卡牌队列**：显示当前队列数量，提供清空按钮

## 七、玩法模块

### 7.1 背包 `InventoryModule`

- 物品列表（名称 / 效果 / 数量）
- 总物品数统计

### 7.2 金币记录 `GoldLogModule`

- 交易日志列表（模块标签 / 原因 / 时间 / 金额增减）
- 当前余额显示

### 7.3 商店 `ShopModule`

- **解锁条件**：需配置第二 API
- 付费刷新（基价 50G + 抽成），携带 generationId 防并发
- 购买即时扣款、物品即时入背包
- 刷新中禁用按钮

### 7.4 工坊 & 交易 `WorkshopModule`（合并工坊 + 股市）

#### 工坊生产

- 工坊等级 1-10，每级 +10% 产出加成
- 选择已解锁角色开始生产
- 按真实时间每秒计算即时增加（离线不算）
- **快速暂停/继续**：暂停时累计已产出，继续时从暂停点继续
- 停止结算：一次性计算差值入账
- 升级消耗：等级 × 200G

#### 风险与机遇（股市）

- **与生产并行**：经营股票不打断生产
- **现金同步**：可支配现金即玩家金币（允许生产收入实时增加可用金额）
- **价格模拟**：离散随机游走 + 轻微均值回归 + 交易冲击
  - 3 秒一次自动变化，交易后立即变化
  - 暂停时折线不变化、无法交易
- **操作**：
  - 买入（+）：按当前价 + 手续费 扣除金币，持仓 +1
  - 卖出（-）：持仓 -1，按当前价 - 手续费 返还金币
  - 可连点快速操作
- **折线图**：
  - 浅色背景 + 水平/垂直网格线 + 价格标签
  - 涨跌用绿色/红色箭头标注
  - 渐变填充（涨绿 / 跌红）
  - 退出（返回）时自动重置
  - 使用 ResizeObserver + nextTick 确保渲染
- **刷新**：随机生成不同初始价格开局
- **暂停/继续**：图标按钮控制
- **退出**：自动平仓卖出全部持仓
- **预计收益**：实时计算（持仓价值 - 手续费 - 已投入）

### 7.5 思绪整理 `Puzzle2048Module`（2048 合成小游戏）

- **点开即玩**：点击卡片自动扣费并开始
- **费用**：金币 ≤ 500 免费，超过按抽成公式计算
- **动态棋盘**：
  - 默认 4×4
  - 4 级工坊解锁 6×6
  - 8 级工坊解锁 8×8
  - 有多个可选大小时显示选择器
- 标准 2048 规则，方块封顶 2048
- 键盘（WASD / 方向键）+ 触屏滑动操作
- **收益**：方块数字之和 × 2048 倍率
  - 0 个 2048: ×1.0
  - 1 个 2048: ×1.5
  - 2 个 2048: ×2.2
  - 3+ 个 2048: ×3.0
- 保存退出：保留棋盘、不结算
- 结算退出：清空棋盘、收益入账

### 7.6 情报交换 `RiddleModule`（AI 对话猜谜）

- **解锁条件**：需配置第二 API
- 用户输入谜底（本地校验，不进入 prompt）
- 用户输入第一条提示开始游戏
- 对话式猜谜，AI 逐轮回复
- **谜底屏蔽**：用户输入包含谜底时阻止发送
- **胜利条件**：AI 回复匹配到谜底
- **奖励**：50 + 轮数 × 20 金币
- 保留上次战绩记录
- **提示词方案**：不使用预设中的 EJS，直接在代码中构造包含角色设定和猜谜规则的提示词
  - 包含示例对话帮助 AI 理解交互模式
  - 支持自定义每轮回复长度建议

### 7.7 末世通讯 `SystemChat`（系统人格聊天）

- **解锁条件**：需配置第二 API
- 与多个系统人格进行普通对话
- 支持 @ 引用楼层内容作为剧情参考
- 支持 Prompt 调试（查看发送给第二 API 的实际提示词）
- **提示词方案**：不使用预设中的 EJS，直接在代码中构造包含角色设定的提示词

### 7.7 经济系统通用规则

- **统一走账**：所有金币变更必须通过 `changeGold(amount, moduleId, reason)`
- **交易日志**：最多保留 50 条记录（模块ID / 原因 / 数额 / 时间戳）
- **背包系统**：`addInventoryItem` 支持堆叠
- **抽成公式**：`calcCommission(gold, workshopLevel, basePrice, threshold=500)`
  - 金币 ≤ 阈值时仅收取基价
  - 超过阈值按工坊等级 0.5%/级收取，封顶 5%
- **股票手续费**：`calcStockFee(workshopLevel)` = workshopLevel × 5，封顶 50
- 所有玩法模块不影响主剧情状态、对话指针、分支锁定
- 系统通知 Toast 永远在最上层（z-index 9999）

### 调试方法

1. **检查Console日志**：
   - 打开浏览器开发者工具
   - 查看Console中的日志信息
   - 搜索 `[ImageGen]`、`[Worldbook]`、`[MessageParser]` 等关键词

2. **检查第二API配置**：
   - 确认URL和Key正确
   - 点击"测试连接"按钮
   - 查看连接状态

3. **检查世界书条目**：
   - 打开世界书管理面板
   - 确认条目已启用
   - 检查API分配和自动控制设置

### 常见问题

**Q: 扇形卡牌队列不显示？**

- 确认同时开启了背景和CG生图开关
- 检查是否已有图片生成
- 确认队列未被清空

**Q: 生图类型判断不准确？**

- 检查消息中是否包含正确的场景标记
- 调整"两者都存在时的优先级"设置
- 查看Console中的判断日志

**Q: 世界书自动控制不工作？**

- 确认世界书条目的关联功能已设置
- 确认自动控制开关已启用
- 检查对应的功能开关（弹幕/生图）是否打开
