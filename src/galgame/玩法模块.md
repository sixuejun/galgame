## 一、经济系统

### EconomyService

- **统一走账**：所有金币变更必须通过 `changeGold(amount, moduleId, reason)`
- **交易日志**：最多保留 50 条记录（模块ID / 原因 / 数额 / 时间戳）
- **背包系统**：`addInventoryItem` 支持堆叠
- **抽成公式**：`calcCommission(gold, workshopLevel, basePrice, threshold=500)`
  - 金币 ≤ 阈值时仅收取基价
  - 超过阈值按工坊等级 0.5%/级收取，封顶 5%
- **股票手续费**：`calcStockFee(workshopLevel)` = workshopLevel × 5，封顶 50

## 二、玩法模块

### 2.1 背包 `InventoryModule`

- 物品列表（名称 / 效果 / 数量）
- 总物品数统计

### 2.2 金币记录 `GoldLogModule`

- 交易日志列表（模块标签 / 原因 / 时间 / 金额增减）
- 当前余额显示

### 2.3 商店 `ShopModule`

- **解锁条件**：需配置第二 API
- 付费刷新（基价 50G + 抽成），携带 generationId 防并发
- 购买即时扣款、物品即时入背包
- 刷新中禁用按钮

### 2.4 工坊 & 交易 `WorkshopModule`（合并工坊 + 股市）

#### 工坊生产

- 工坊等级 1-10，每级 +10% 产出加成
- 选择已解锁角色开始生产
- 按真实时间每秒计算即时增加（离线不算）
- **快速暂停/继续**：暂停时累计已产出，继续时从暂停点继续
- 停止结算：一次性计算差值入账
- 升级消耗：等级 × 200G

#### 风险与机遇（股市）

- **与生产并行**：经营股票不打断生产
- **现金同步**：可支配现金即玩家金币（允许生产收入实时增加可用金额）
- **价格模拟**：离散随机游走 + 轻微均值回归 + 交易冲击
  - 3 秒一次自动变化，交易后立即变化
  - 暂停时折线不变化、无法交易
- **操作**：
  - 买入（+）：按当前价 + 手续费 扣除金币，持仓 +1
  - 卖出（-）：持仓 -1，按当前价 - 手续费 返还金币
  - 可连点快速操作
- **折线图**：
  - 浅色背景 + 水平/垂直网格线 + 价格标签
  - 涨跌用绿色/红色箭头标注
  - 渐变填充（涨绿 / 跌红）
  - 退出（返回）时自动重置
  - 使用 ResizeObserver + nextTick 确保渲染
- **刷新**：随机生成不同初始价格开局
- **暂停/继续**：图标按钮控制
- **退出**：自动平仓卖出全部持仓
- **预计收益**：实时计算（持仓价值 - 手续费 - 已投入）

### 2.5 思绪整理 `Puzzle2048Module`（2048 合成小游戏）

- **点开即玩**：点击卡片自动扣费并开始
- **费用**：金币 ≤ 500 免费，超过按抽成公式计算
- **动态棋盘**：
  - 默认 4×4
  - 4 级工坊解锁 6×6
  - 8 级工坊解锁 8×8
  - 有多个可选大小时显示选择器
- 标准 2048 规则，方块封顶 2048
- 键盘（WASD / 方向键）+ 触屏滑动操作
- **收益**：方块数字之和 × 2048 倍率
  - 0 个 2048: ×1.0
  - 1 个 2048: ×1.5
  - 2 个 2048: ×2.2
  - 3+ 个 2048: ×3.0
- 保存退出：保留棋盘、不结算
- 结算退出：清空棋盘、收益入账

### 2.6 情报交换 `RiddleModule`（AI 对话猜谜）

- **解锁条件**：需配置第二 API
- 用户输入谜底（本地校验，不进入 prompt）
- 用户输入第一条提示开始游戏
- 对话式猜谜，AI 逐轮回复
- **谜底屏蔽**：用户输入包含谜底时阻止发送
- **胜利条件**：AI 回复匹配到谜底
- **奖励**：50 + 轮数 × 20 金币
- 保留上次战绩记录
- **提示词方案**：不使用预设中的 EJS，直接在代码中构造包含角色设定和猜谜规则的提示词
  - 包含示例对话帮助 AI 理解交互模式
  - 支持自定义每轮回复长度建议

## 三、废土行路

### 3.1 已实现的功能

#### 鼠标移动到地图边缘时自动移动视图

**功能描述：**
- 当鼠标靠近地图视图边缘（80像素范围内）时，会显示方向箭头提示
- 鼠标悬停在箭头上时，视图会自动向该方向平移
- 支持上下左右四个方向的自动滚动

**实现细节：**
- 添加了 4 个方向箭头组件（上、下、左、右）
- 使用 `handleMouseMove` 函数检测鼠标位置
- 边缘阈值：80 像素
- 滚动速度：每帧 8 像素
- 箭头带有呼吸动画效果，提升视觉体验

#### 投掷骰子后自动移动

**功能描述：**
- 投掷骰子后，如果只有一条路径可走，系统会自动移动
- 遇到分叉路口（多条路径）时停下，等待玩家选择
- 自动移动过程中会显示剩余步数

**逻辑流程：**
1. 玩家掷骰子
2. 系统检查可走路径数量
3. 如果只有 1 条路径 → 自动移动
4. 如果有 2 条或以上路径 → 停下等待玩家选择
5. 重复步骤 2-4 直到步数用完

#### 步数走完之前不进入随机事件

**功能描述：**
- 只有在步数完全用完（`stepsRemaining <= 0`）时才会触发事件
- 在自动移动过程中，即使经过事件格子也不会触发
- 确保玩家能完整走完骰子点数

#### 视图自动跟随玩家

**额外优化功能：**
- 玩家移动时，视图会平滑地跟随玩家位置
- 使用 GSAP 动画库实现流畅的视图过渡
- 在动画过程中实时更新视口位置
- 移动完成后自动居中到玩家位置

### 3.2 配置参数

可以根据需要调整以下参数：

```typescript
const EDGE_THRESHOLD = 80  // 边缘触发距离（像素）
const PAN_SPEED = 8        // 滚动速度（像素/帧）
const AUTO_MOVE_DELAY = 600 // 自动移动延迟（毫秒）
```

## 四、跨模块共通规则

- 所有玩法模块不影响主剧情状态、对话指针、分支锁定
- 金币/资源统一进入 EconomyService
- 同一时间最多打开一个玩法模块
- 退出工坊前若正在生产，阻止关闭（需先停止结算或暂停）
- 系统通知 Toast 永远在最上层（z-index 9999）
