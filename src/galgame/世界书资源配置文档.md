# 消息解析系统重构 - 世界书资源配置文档

## 📚 世界书资源定义格式

### 1. 背景资源

在世界书中创建一个条目，内容为 JSON 格式：

```json
{
  "type": "background",
  "backgrounds": [
    {
      "name": "废墟",
      "file": "https://example.com/bg_ruins.jpg",
      "textMappings": ["废墟", "荒地", "废弃建筑", "破败的街道"]
    },
    {
      "name": "家中",
      "file": "https://example.com/bg_home.jpg",
      "textMappings": ["家里", "房间", "卧室", "客厅"]
    },
    {
      "name": "酒馆",
      "file": "https://example.com/bg_tavern.jpg",
      "textMappings": ["酒馆", "酒吧", "小酒馆"]
    }
  ]
}
```

**字段说明**：
- `type`: 固定为 `"background"`
- `backgrounds`: 背景资源数组
  - `name`: 背景名称（用于精确匹配）
  - `file`: 背景图片URL
  - `textMappings`: 可选，文本映射数组，用于模糊匹配

### 2. CG资源

```json
{
  "type": "cg",
  "cgs": [
    {
      "name": "拥抱",
      "file": "https://example.com/cg_hug.jpg",
      "textMappings": ["拥抱", "抱住", "紧紧相拥", "相拥而泣"]
    },
    {
      "name": "告别",
      "file": "https://example.com/cg_farewell.jpg",
      "textMappings": ["告别", "离别", "道别", "分离"]
    }
  ]
}
```

**字段说明**：
- `type`: 固定为 `"cg"`
- `cgs`: CG资源数组
  - `name`: CG名称
  - `file`: CG图片URL
  - `textMappings`: 可选，文本映射数组

### 3. 立绘资源

```json
{
  "type": "sprite",
  "sprites": [
    {
      "name": "程北极",
      "file": "https://example.com/sprite_cheng.png",
      "textMappings": ["程北极", "北极", "小程"]
    },
    {
      "name": "白娅",
      "file": "https://example.com/sprite_baiya.png",
      "textMappings": ["白娅", "小白", "娅娅"]
    }
  ]
}
```

**字段说明**：
- `type`: 固定为 `"sprite"`
- `sprites`: 立绘资源数组
  - `name`: 角色名称
  - `file`: 立绘图片URL
  - `textMappings`: 可选，角色名称的其他叫法

## 📝 消息格式说明

### 1. 角色对话

```
[[character||角色名：程北极||场景：废墟||动作：抱胸||表情：生气||台词：你怎么这样！]]
```

**字段**：
- `角色名`: 角色名称（会匹配立绘资源）
- `场景`: 场景名称（会匹配背景资源）
- `动作`: 动作描述（预留字段）
- `表情`: 表情描述（预留字段）
- `台词`: 角色说的话

### 2. CG场景

```
[[character||角色名：程北极||CG场景：拥抱||台词：终于找到你了...]]
```

**特点**：
- 使用 `CG场景` 字段代替 `场景`
- CG显示时会隐藏背景和立绘
- 生成的图片也会显示在CG层

### 3. 旁白

```
[[narration||场景：废墟||旁白：天色渐暗，废墟中传来奇怪的声音。]]
```

**字段**：
- `场景`: 场景名称（可选，会匹配背景资源）
- `旁白`: 旁白内容

### 4. 黑屏文字

```
[[blacktext||黑屏文字：三天后...]]
```

**特点**：
- 全屏黑色背景
- 文字居中显示
- 适合用于时间跳跃、章节分隔等

### 5. 用户消息

```
[[user||场景：废墟||台词：我们快走吧！]]
```

**字段**：
- `场景`: 场景名称（可选）
- `台词`: 用户说的话

## 🎯 场景继承机制

如果某个消息块没有指定场景，会自动继承上一个消息块的场景：

```
[[character||角色名：程北极||场景：废墟||台词：这里很危险。]]
[[character||角色名：白娅||台词：我知道。]]
```

第二个消息块会自动继承"废墟"场景。

---

## 🔍 模糊匹配规则

系统会对场景名、角色名进行模糊匹配：

1. **去除特殊符号**：`-`、`/`、`_`、`.`、空格等
2. **转为小写**
3. **包含匹配**：只要包含关键词即可

**示例**：
- 消息中的 `废-墟` 可以匹配到 `废墟`
- 消息中的 `程 北极` 可以匹配到 `程北极`
- 消息中的 `小程` 可以通过 `textMappings` 匹配到 `程北极`

---

## 🎨 分层结构

系统使用以下分层结构（从上到下）：

1. **UI层** (z-index: 20) - 对话框、菜单等UI元素
2. **弹幕层** (z-index: 5) - 弹幕显示
3. **CG层** (z-index: 4) - CG图片和生成的图片
4. **效果层** (z-index: 3) - 暗角、噪点等视觉效果
5. **立绘层** (z-index: 2) - 角色立绘
6. **背景层** (z-index: 1) - 背景图片

**注意**：
- CG显示时，立绘和背景会被隐藏
- 生成的图片根据类型显示在背景层或CG层

## 🐛 调试技巧

1. **查看控制台日志**：
   - `[WorldbookLoader]` - 世界书资源加载日志
   - `[MessageParser]` - 消息解析日志
   - `[消息解析]` - 消息解析状态

2. **常见问题**：
   - 资源未匹配：检查 `textMappings` 是否包含消息中的文本
   - 图片不显示：检查图片URL是否可访问
   - 场景未继承：确保上一个消息块有场景字段

3. **清除缓存**：
   - 切换聊天会自动清除缓存
   - 资源缓存时间为5秒

---

## 📌 注意事项

1. **JSON格式**：世界书条目内容必须是有效的JSON格式
2. **图片URL**：确保图片URL可访问，支持HTTPS
3. **字段名称**：支持中英文冒号（`:` 和 `：`）
4. **向后兼容**：如果消息中没有新格式，会降级为纯文本显示
5. **性能优化**：世界书资源会缓存5秒，避免频繁加载
